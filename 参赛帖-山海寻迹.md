## 一句话

**你的家乡有没有一只连你都不知道的守护神？**

---

## 我们解决的问题

《山海经》是中国最古老的神话地理志，记载了 104 只上古神兽，分布在 34 个省份。但两千年来，没有人真正让它们**开口说话**——直到现在。

现存的所有《山海经》数字产品，要么是静态百科，要么是电子书翻页。缺少的是**让古籍"活起来"的 AI 体验**。

---

## 我们的方案

**山海寻迹**是一个 AI 驱动的《山海经》神兽探索平台——让 104 只上古神兽在 34 种方言中复活：

| 功能 | 体验 |
|------|------|
| 🗺️ **34 省交互地图** | 暗黑水墨风格中国地图，点击省份→迷雾散开→神兽遭遇 |
| 🔊 **方言 TTS 朗读** | 每只神兽用自己省份的方言朗读《山海经》古文原文——毕方用陕西话、九尾狐用湖南话、应龙用河南话 |
| 💬 **AI 神兽对话** | 可与神兽进行角色化 NPC 对话，每只神兽有独立性格设定与羁绊记忆 |
| 🛡️ **家乡守护神** | 基于浏览器定位匹配用户省份→抽取专属守护神→生成分享海报 |
| 📜 **卷册图鉴** | 按《山海经》原书结构分五卷（南山/西山/北山/东山/中山+海外大荒），节气限时收集 |

---

## AI 亮点

项目部署了**三层 AI 架构**，用户在每个交互节点都能感知到 AI 在工作：

### 第一层：LLM 神兽角色对话
- 每只神兽有独立的 System Prompt（性格、说话风格、知识边界）
- 三 Provider 容错架构（DeepSeek → MiMo → DashScope 自动降级）
- 打字机效果 + 羁绊记忆注入（神兽记得你送过什么、聊过什么）
- 离线规则回退（无 API Key 时古文翻译式回复）

### 第二层：方言 TTS 语音合成
- 基于阿里云 CosyVoice V2 API，104 只神兽全部生成方言语音
- **34 省方言全覆盖**：四川话、河南话、粤语、东北话、藏语、蒙古语…
- 3 级降级：预生成 MP3 → API 实时合成 → Web Speech API 回退

### 第三层：AI 见闻录生成
- 根据用户收集路径自动生成文言文探险手记
- 4 阶段动态生成：初见→收录→羁绊→节气

### 技术栈
React 19 + TypeScript 6 + Vite 7 + Three.js + D3 Canvas + GSAP + Zustand · DashScope CosyVoice · DeepSeek/MiMo/DashScope LLM

## 社会价值

### 古籍活化
《山海经》是中华文化的源头性文本，但对其的认知长期停留在"儿童读物"层面。我们通过 AI 对话 + 方言朗读 + 交互地图的组合，让用户像"探索未知世界"一样进入古籍——**不是被动阅读，而是主动发现**。

### 方言保护
中国 34 个省级行政区有数百种方言，但年轻一代使用能力急剧下降。我们让每只神兽用家乡方言朗读古文——用户在收藏神兽的同时，也在"收藏"即将消失的声音。**听毕方用陕西话念《山海经》，听九尾狐用湖南话讲青丘之山——这是只有 AI 时代才能做到的文化保护**。
